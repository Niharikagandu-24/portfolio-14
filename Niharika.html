Design and Implementation of Low-Power Approximate Multipliers
Mar 2025 • SRM EPIC 2025
▶

8-bit approximate multiplier using a novel 4:2 compressor with error correction for low-power image processing.

Approximate Computing 4:2 Compressor Vivado FPGA (Nexys-4 DDR)
Implemented a low-power 8-bit approximate multiplier based on a customized 4:2 compressor and error-correction logic. The design significantly reduced hardware complexity while keeping the error rate < 0.5%. Achieved around 9% reduction in dynamic power, 87% reduction in static power, and 31% LUT reduction compared to the accurate baseline multiplier on FPGA. Validated the design using image processing workloads with high SNR/PSNR, showing it is suitable for battery-operated and IoT applications.
4:2 Compressor-Based Approximate Multipliers for Image Processing
Oct 2025 • Ongoing
▶

Exploring 4:2 compressor-based approximate multiplier architectures optimized for image quality vs power.

Image Processing MATLAB Approximate Arithmetic FPGA Implementation
Designing a family of 4:2 compressor-based approximate multipliers and validating their performance on image processing tasks using a MATLAB–to–Vivado flow. Evaluating metrics such as PSNR (> 45 dB), mean error distance, and hardware utilization to identify optimal trade-offs between power savings and image quality. Targeting low-power embedded and portable vision systems where slight inaccuracy is acceptable.
32-bit Wallace Tree Multiplier – Design, Synthesis & Physical Implementation
Jun 2025
▶

High-speed 32-bit Wallace Tree multiplier implemented through the complete RTL-to-GDSII digital flow.

Wallace Tree Synopsys Design Compiler IC Compiler II Static Timing Analysis
Developed a 32-bit Wallace Tree Multiplier in Verilog and verified functionality using Synopsys VCS. Performed logic synthesis with Design Compiler, followed by floorplanning, power planning, placement, clock tree synthesis, and routing using IC Compiler II. Analyzed area, timing, and power trade-offs and iterated to close timing under given constraints. This project gave end-to-end exposure to the digital ASIC RTL-to-GDSII flow.
